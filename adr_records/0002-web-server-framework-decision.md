# 2. Выбор веб-фреймворка

Дата: 2025-11-29

## Статус

Принято

## Контекст

Требуется выбрать веб-фреймворк для ML-сервиса, в котором будет доступен REST API. При этом нужно учитывать то, что приложение будет работать в контейнере.

## Решение

Варианты:
* FastAPI - плюсы: production-ready, высокая производительность; минусы: зависит от pydantic, НЕ легковесный.
* Django - плюсы: готовый функционал, прост в использовании; минусы: тяжеловесный, относительно низкая производительность.
* Flask - плюсы: легковесный, высокая производительность; минусы: требует запуска в wsgi для production-окружения.

Выбор: Используем Flask, так как требуются минимальный вес приложения, высокая производительность и простота использования. Django в данном случае избыточен, так как нам не требуются его возможности вроде встроенного ORM.

Компромиссы: чуть сложнее написание кода, но архитектура приложения проще.

## Последствия

Используется python3+flask.
