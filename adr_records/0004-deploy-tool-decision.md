# 4. Выбора инструмента развёртки.

Дата: 2025-11-29

## Статус

Принято

## Контекст

Хоть приложение и в контейнере, этот контейнер нужно разворачивать на машину с docker daemon.

## Решение

Варианты:
* Ansible - плюсы: возможность глубокой настройки хоста, соблюдение IaC; минусы: подходит только для хостов, то есть не для serverless сервисов.
* Terraform - плюсы: совместимость с cloud-сервисами; минусы: настройка хоста через него неэффективна.

За основу взято прошлое [решение](https://github.com/nasibulinaa/mipt_ml_2024/tree/main/docker) по сборке контейнера.

Вывод: terraform будет достаточно, так как разработка и запуск приложения происходят в рамках MVP, не требуется сложного решения. 

Компромиссы: сложнее Ansible, для развёртки на хост нужно будет перейти на другое решение, но будет работать с serverless-сервисом.

## Последствия

Используем terraform + docker provider для развёртки приложения.
